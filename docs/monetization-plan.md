# SetupRadar Monetization Policy & Layout Plan

이 문서는 Project7 SetupRadar에 광고와 제휴 링크를 나중에 붙일 때 지켜야 할 기준을 정리한다. 현재 단계에서는 실제 AdSense 코드, 광고 script, 링크프라이스/쿠팡파트너스 링크, 제품 구매 버튼을 추가하지 않는다.

참고한 공식 기준:

- Google AdSense Program policies: https://support.google.com/adsense/answer/48182
- Google AdSense site readiness guide: https://support.google.com/adsense/answer/7299563
- Google AdSense policy FAQ: https://support.google.com/adsense/answer/3394713

## 1. 수익화 기본 원칙

SetupRadar는 광고 사이트가 아니라 하드웨어 테스트 도구와 구매 전 참고 가이드 사이트다. 수익화는 이 정체성을 해치지 않는 범위에서만 적용한다.

- 테스트 도구와 Finder의 신뢰도를 광고 수익보다 우선한다.
- Finder 추천 결과는 제휴 수익이 아니라 `basicFilters`, 구매 전 체크, 제품 데이터 품질을 기준으로 유지한다.
- 광고와 제휴 링크는 사용자에게 명확히 표시한다.
- 광고 배치는 테스트 실행, 결과 확인, Finder 조건 선택을 방해하지 않아야 한다.
- 불확실한 제품 스펙, 가격, 판매 옵션은 확정 표현하지 않는다.
- 광고 클릭을 유도하거나 광고를 콘텐츠처럼 오해하게 만드는 문구를 쓰지 않는다.
- 제휴 링크가 있는 제품과 없는 제품이 추천 점수에 영향을 주지 않는다.

## 2. 수익화 우선순위

### 1순위: Google AdSense 방문형 광고

적용 후보:

- 메인
- 테스트 목록
- 가이드 목록
- 테스트 상세
- 가이드 상세

원칙:

- 첫 화면 Hero 바로 아래에는 두지 않는다.
- 테스트 실행 버튼, Finder 질문 선택 영역, 결과 조작 영역 주변에는 두지 않는다.
- 광고 클릭을 유도하는 문구를 쓰지 않는다.
- AdSense 승인 전에는 광고 자리 표시를 과하게 만들지 않는다.

### 2순위: 제품 관련 제휴 링크

후보:

- 링크프라이스
- 쿠팡파트너스
- 향후 해외 제휴

적용 후보:

- Finder 결과 카드
- 구매 전 체크 패널
- 향후 제품 상세 페이지

원칙:

- 바로 구매를 압박하는 `구매하기`보다 `가격/옵션 확인`, `공식 스펙 확인`, `국내 AS 조건 확인` 톤을 우선한다.
- `review` 상태 제품에는 제휴 링크를 자동 노출하지 않는다.
- `published` 제품이라도 링크 자체는 별도 `approved` 상태를 거친 뒤 노출한다.

### 3순위: 가격 확인 / 공식 스펙 확인 링크

원칙:

- 최저가 보장 표현을 쓰지 않는다.
- 가격 추적이나 자동 크롤링처럼 운영 부담이 큰 기능은 뒤로 미룬다.
- 링크 문구는 `가격 확인`, `판매처 옵션 확인`, `공식 스펙 확인` 정도로 제한한다.

### 보류

- 리뷰 영상 embed
- YouTube 썸네일/영상 임베드
- 실시간 가격 트래커
- 자동 크롤링
- 제품 비교 대시보드
- 유료 리포트
- 다국어 광고 분기

## 3. AdSense 슬롯 후보 위치

이 섹션은 위치 후보만 정리한다. 실제 슬롯 컴포넌트나 광고 코드는 만들지 않는다.

### 메인 `/kr`

- Hero 바로 아래는 피한다.
- 테스트 도구 섹션 아래 1개 후보.
- 가이드 섹션 아래 1개 후보.
- 첫인상이 광고판처럼 보이지 않게 페이지 상단 광고 밀도를 낮춘다.

### Tests 목록 `/kr/tests`

- 테스트 목록 상단 설명 아래 1개 후보.
- 목록 하단 1개 후보.
- 테스트 카드 사이에 반복 삽입하지 않는다.

### Test 상세

- 설명/주의사항 이후 1개 후보.
- 결과 영역 아래 1개 후보.
- 테스트 시작, 초기화, 결과 버튼 주변에는 광고를 두지 않는다.
- 오작동이나 오클릭을 유도하는 배치를 금지한다.

### Guides 목록 `/kr/guides`

- 목록 상단 설명 아래 1개 후보.
- 가이드 카드 목록 하단 1개 후보.
- 카테고리 제목과 카드 사이에는 넣지 않는다.

### Guide 상세

- 본문 중간 1개 후보.
- 본문 하단 1개 후보.
- 문단마다 광고를 끼워 넣지 않는다.
- 짧은 가이드에는 중간 광고를 생략한다.

### Finder

- 질문 선택 영역에는 광고를 두지 않는다.
- 결과 카드 사이에는 광고를 두지 않는다.
- 결과 전체 하단에만 광고 후보를 둔다.
- 구매 전 체크 패널 내부에는 광고보다 공식/제휴 확인 링크 후보를 우선 검토한다.

## 4. 제휴 링크 후보 위치

### Finder 결과 카드

- 기본 카드에서는 제품명, 요약, 핵심 기준, 체감 한줄평을 우선한다.
- 제휴 링크는 하단 작은 영역으로 보류한다.
- 문구 후보:
  - `가격/옵션 확인`
  - `판매처 옵션 확인`
  - `공식 스펙 확인`
- `review` 상태 제품에는 노출하지 않는다.

### 구매 전 체크 패널

- 구매를 압박하지 않고 확인 용도 중심으로 둔다.
- 문구 후보:
  - `판매처 옵션 확인`
  - `공식 스펙 확인`
  - `국내 AS 조건 확인`
- 스펙과 판매 옵션이 달라질 수 있다는 안내를 같이 둔다.

### 향후 제품 상세 페이지

권장 구조:

1. 제품 요약
2. Finder 기준과 핵심 스펙
3. 구매 전 체크
4. 상세 스펙
5. 가격/옵션 확인 링크
6. 제휴 고지 문구

제품 상세 페이지가 생기기 전에는 Finder 결과 카드에 제휴 링크를 과하게 넣지 않는다.

## 5. 제휴/광고 고지 문구 후보

사이트 공통 또는 제휴 링크가 있는 영역에 아래 문구를 상황에 맞게 사용한다.

- 이 페이지에는 광고 또는 제휴 링크가 포함될 수 있습니다.
- 일부 링크를 통해 구매가 발생하면 운영자가 일정 수수료를 받을 수 있습니다.
- 사용자에게 추가 비용이 발생하지 않으며, 제품 선택 기준은 구매 전 체크 기준을 우선합니다.
- 가격과 옵션은 판매처에 따라 달라질 수 있으니 구매 전 직접 확인해 주세요.

금지 표현:

- 최저가 보장
- 공식 최저가
- 무조건 추천
- 반드시 구매
- 지금 바로 사야 함
- 후회 없음

## 6. AdSense 신청 전 체크리스트

- 개인정보처리방침 페이지가 필요하다.
- 이용약관 또는 서비스 안내 페이지가 필요하다.
- 문의 채널이 필요하다.
- 사이트 목적이 첫 화면에서 명확해야 한다.
- 복사 콘텐츠보다 자체 도구와 자체 가이드가 보여야 한다.
- 준비 중 링크가 너무 많으면 줄여야 한다.
- 빈 페이지와 미구현 링크를 최소화해야 한다.
- 테스트 도구가 실제로 작동해야 한다.
- 모바일 화면에서 광고 후보 위치가 콘텐츠 조작을 방해하지 않아야 한다.
- 광고 클릭 유도 문구가 없어야 한다.
- Search Console과 sitemap 상태는 계속 추적하되, sitemap 제출 지연은 코드 문제가 확인될 때만 수정한다.

## 7. 제휴 링크 적용 전 체크리스트

- 제품 데이터가 일정 수준 이상 쌓였는지 확인한다.
- `review` 제품과 `published` 제품의 공개 정책을 확정한다.
- `productLinks.status` 또는 이에 준하는 승인 상태를 둔다.
- 제휴 링크가 있는 제품과 없는 제품이 추천 점수에 영향을 주지 않는지 확인한다.
- 링크가 깨졌을 때 숨김/보류 처리 기준을 둔다.
- 링크프라이스/쿠팡파트너스 고지 문구를 화면에 넣는다.
- 제품 가격과 옵션이 바뀔 수 있다는 안내를 둔다.
- 제휴 링크 클릭 추적은 GA4 이벤트로 나중에 검토한다.

## 8. 구현 단계 로드맵

### Phase 0: 현재 단계

- 수익화 정책과 위치 후보만 문서화한다.
- 광고/제휴 코드 없음.
- 제품 링크 없음.

### Phase 1: 정책 페이지 준비

- 개인정보처리방침
- 이용약관 또는 서비스 안내
- 문의 채널
- 광고/제휴 고지 문구

### Phase 2: AdSense 신청 준비

- 콘텐츠/가이드/도구 품질 보강
- 미구현 링크 정리
- 도메인 구매 여부 검토
- AdSense 신청

### Phase 3: 광고 슬롯 컴포넌트

- 실제 광고 코드 없이 placeholder 컴포넌트 설계.
- 환경변수로 켜고 끌 수 있게 설계.
- 광고 미승인 상태에서는 렌더링하지 않음.
- 테스트 실행 버튼, Finder 질문, 결과 조작 영역 주변에는 렌더링 금지.

### Phase 4: 제휴 링크 구조

- `productLinks` 구조 확장.
- `review` 제품 자동 노출 금지.
- `published` 제품 + `approved` 링크만 노출.
- Finder 결과 카드에는 과하지 않게 표시.
- 제품 상세 페이지가 생기면 상세 페이지 하단에 우선 배치.

### Phase 5: GA4 이벤트

후보 이벤트:

- `affiliate_click`
- `finder_result_expand`
- `guide_cta_click`
- `test_start`
- `test_complete`

현재는 구현하지 않는다.

## 9. 하지 말아야 할 것

- 광고 먼저 붙이기.
- AdSense 승인 전 광고 자리 도배.
- Finder 결과를 제휴 수익 기준으로 정렬.
- 가격을 크롤링해서 최저가처럼 표시.
- 리뷰 영상이나 썸네일 무단 사용.
- 제품 추천을 확정적으로 표현.
- 개인정보처리방침 없이 광고/분석/제휴를 과하게 붙이기.
- 테스트 버튼 주변에 광고를 둬서 오클릭을 유도하기.
- 광고를 콘텐츠 카드처럼 보이게 만들기.

## 10. 현재 판단

지금은 AdSense나 제휴 코드를 붙일 단계가 아니다. SetupRadar는 먼저 자체 도구, 구매 전 체크 가이드, Finder의 신뢰도를 안정화해야 한다.

우선순위:

1. 정책 페이지 준비.
2. 문의 채널 정리.
3. 콘텐츠 품질 보강.
4. Search Console sitemap 상태 추적.
5. 도메인 구매 여부 검토.
6. AdSense 신청.
7. 광고 슬롯과 제휴 링크를 환경변수/승인 상태 기반으로 제한 노출.

도메인은 아직 바로 구매하지 않아도 된다. 다만 AdSense 신청 전에는 커스텀 도메인을 검토할 수 있다.

## 11. Policy Pages Prep 상태

2026-05-30 기준 최소 정책/안내 페이지를 먼저 준비했다.

- `/kr/privacy`: 개인정보처리방침
- `/kr/terms`: 이용 안내
- `/kr/disclosure`: 광고·제휴 고지

이 페이지들은 AdSense와 제휴 링크를 바로 붙이기 위한 구현이 아니라, 사용자 신뢰와 심사 준비를 위한 기본 안내다. 실제 광고 script, AdSense 코드, 제휴 링크, 구매 버튼은 아직 추가하지 않았다.

## 12. Project2 연동 방향 메모

2026-06-07 기준 Project7 메인에는 정적 mock 데이터 기반의 최근 장비 소식 preview만 추가했다. 실제 Project2 연동, RSS/API/DB 연동, 자동 수집, 외부 뉴스 링크, 제휴 링크는 아직 구현하지 않는다.

향후 역할 분리는 다음 방향으로 검토한다.

- Project7: 테스트 도구, 구매 전 확인 기준, Finder, 제품 후보 비교 흐름을 담당한다.
- Project2: 핫딜, 뉴스, 제품 소개형 본문 콘텐츠를 담당할 수 있다.
- Project7 메인: Project2 글 전체를 가져오기보다 검수된 preview 카드만 노출하는 방향을 우선 검토한다.
- 제휴나 광고가 붙는 글을 preview할 때는 광고·제휴 고지와 링크 상태를 먼저 확인한다.
- 모든 글을 한 번에 연동하지 않고, 자주 보는 장비 이슈와 내부 가이드로 연결되는 글부터 단계적으로 검토한다.

## 13. VS Compare와 Project2 연동 후보

2026-06-07 기준 VS Compare는 구조 계획만 문서화했다. 실제 비교 route, 비교 상세 페이지, CPU/GPU 데이터, Project2 연동은 아직 구현하지 않는다.

향후 비교 콘텐츠와 수익화/Project2 역할은 다음 방향으로 검토한다.

- Project7 비교 콘텐츠는 장비 선택 기준, 호환성, 구매 전 확인, Finder와 내부 가이드 연결을 우선한다.
- Project2는 제품 소개 본문, 핫딜, 뉴스, 구매처 정보 후보를 담당할 수 있다.
- Project7 비교 페이지 하단에는 검수된 Project2 관련 글 preview만 노출하는 구조를 장기 후보로 둔다.
- 실제 링크프라이스/쿠팡파트너스 링크는 Project2 쪽에서 먼저 관리하고, Project7 비교 결론이나 추천 순서에는 영향을 주지 않는 방향이 안전하다.
- CPU/GPU 비교는 벤치마크 수치 복제보다 공식 스펙, 플랫폼 호환성, 전력/발열, 업그레이드 경로, 측정 환경 차이 안내를 우선한다.

수익화 관점의 연결 원칙:

- Project7은 제품 후보를 고르는 기준, Finder, Compare, 테스트 도구를 우선한다.
- Project2는 뉴스 본문, 제품 소개, 핫딜, 구매처/배송대행/구매대행 소개, 제휴 링크 후보를 맡는 방향을 검토한다.
- Project7에 Project2 preview를 붙이더라도 검수된 글만 수동 연결하는 단계부터 시작한다.
- 제휴 링크는 Finder/Compare 추천 점수, 비교 결론, 카드 정렬과 분리한다.
- AdSense 심사 전에는 상업성보다 자체 도구, 가이드, 비교 기준의 독립 가치를 우선한다.
